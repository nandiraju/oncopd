# OncOPD — AI-Powered Precision Oncology OPD

A **frontend-only, local demo prototype** of an oncologist's daily command center, evolved from Dr.G. Premium healthcare-SaaS UI (dark navy + warm gold, 1Cell.Ai inspired). No backend, no database, no external APIs, no real patient data.

> ⚠️ Synthetic demo data only. Frontend-only local prototype. Not for clinical use.

## Run

```bash
npm install
npm run dev      # http://localhost:5173  (regenerates synthetic data first)
```

Other scripts: `npm run build` (typecheck + production build), `npm run seed` (regenerate `/src/data`).

## Stack

- **React 18 + TypeScript + Vite**
- **Tailwind CSS** with shadcn-style hand-built UI primitives (`src/components/ui`)
- **Zustand** + `persist` middleware → **localStorage** for demo persistence
- **React Router** for navigation
- Static synthetic JSON in `/src/data`, deterministically generated by `scripts/generate-data.mjs`

## Datasets (`/src/data`)

100 patients · 250 appointments · 150 reports · 364 timeline events · 200 orders · 150 prescriptions · 300 OSakhi messages · 50 peer reviews · 40 MTB cases · 30 oncologists · 20 clinics. Covers 12 cancer types and biomarkers (EGFR exon 19 del, L858R, ALK/ROS1 fusion, KRAS G12C, BRAF V600E, MET exon 14, BRCA1/2, MSI-high, TMB-high, PD-L1 TPS, HER2 amp, HRD+).

## Screens

1. **Dashboard / Daily Worklist** — stat cards, filterable worklist (Today, Urgent, Reports Pending, Follow-up Due, New Diagnosis, Recurrence, MTB Needed), next-up rail, critical findings, distress alerts.
2. **Patient Registry** — searchable/filterable/sortable table.
3. **Single Patient View** — 12 tabs: Overview, Timeline, Reports, Molecular Profile, Treatment History, Orders, Prescriptions, Appointments, OSakhi, MTB, Notes, ChatEMR.
4. **AI Chart Review** — 5 rule-based actions (60-sec summary, what changed, open questions, patient-friendly, MTB-ready).
5. **ChatEMR** — patient-specific mock chat, intent-routed answers, persisted history.
6. **Order Wizard** — 5-step guided flow with AI-suggested tests per scenario.
7. **Report Wizard** — genomics/pathology/IHC/liquid biopsy/germline/radiology with 8 interpretation sections.
8. **Prescription & Care Plan** — regimen, cycles, supportive care, monitoring, OSakhi reminder toggle.
9. **Appointments** — today/upcoming/teleconsult/report-review/missed + reschedule & OSakhi reminders.
10. **OSakhi Inbox** — messages, AI-suggested replies, escalate/convert/add-to-note actions.
11. **Peer Review / Second Opinion** — create & track requests with AI case briefs.
12. **Personalized MTB** — full case pages + presentation-ready "Generate MTB Brief" (exportable).
13. **Settings** — local-data counts + **Reset Demo Data**.

## AI

All "AI" output is **rule-based and generated locally** from the synthetic data (`src/lib/mock-ai.ts`) and clearly labeled as synthetic demo content. No LLM/API calls.

## Persistence

Created orders, OSakhi replies, chart notes, peer-review requests, ChatEMR history, MTB briefs, reminder toggles and appointment status are saved to `localStorage` (`oncopd-demo-v1`). **Settings → Reset Demo Data** clears them and reloads the original JSON.
